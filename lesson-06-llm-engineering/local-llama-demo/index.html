🍎 Чому Apple Silicon "виняток з правил" для LLM?

🖥️ Традиційний ПК (Discrete GPU)

CPU має свою RAM (32GB), GPU має свою VRAM (8GB)

❌ Проблема: Якщо модель 10GB, а VRAM 8GB — не влізе

⚠️ Bottleneck: Дані постійно пересилаються через PCIe → повільно

RTX 4060 (8GB) + 32GB RAM:
Llama 70B = 1 токен/сек (на CPU)

🍎 Apple Silicon (Unified Memory)

CPU, GPU, Neural Engine — на одному кристалі, спільна пам'ять

✅ Рішення: Якщо 32GB RAM → GPU може використати майже все

⚡ Преимущество: Немає копіювання, прямий доступ до пам'яті

MacBook M2 Pro (32GB):
Llama 70B = 5-8 токенів/сек ✨

🔧 Технічні деталі

Memory Bandwidth (швидкість доступу до пам'яті):

  • RTX 4090: 960 GB/s
  • M2 Max: 400 GB/s (менше, але немає bottleneck на копіювання)
  • M3 Max: 800 GB/s (приблизно як RTX 4090)

💡 На Apple Silicon ви економите час на копіюванні 10GB моделі туди-сюди, що на практиці дає більше користі, ніж просто raw bandwidth

📊 Практичний приклад

Модель RTX 4060 (8GB VRAM) MacBook M2 Pro (32GB)
Llama 3.2 1B 30 токенів/сек ✓ 50+ токенів/сек ✨
Llama 3 8B 15 токенів/сек ✓ 20-25 токенів/сек ✓
Llama 3 70B ❌ Not possible (40GB RAM потребує) 5-8 токенів/сек 🎉
Llama 3.1 405B ❌ Impossible ❌ Потребує 192GB+ (Mac Studio)

🎯 Ключові висновки

Unified Memory ≠ Shared Video Memory
Це не старі вбудовані карти Intel. Це професійна швидка архітектура на одному кристалі
Для AI-інженера сьогодні:
MacBook з 32GB+ RAM — наймобільніша та найпотужніша лабораторія для локальних LLM
Порада для розробників:
Навіть MacBook Air M1 з 8GB зможе запустити Llama 1B з прийнятною швидкістю. Спробуйте!
