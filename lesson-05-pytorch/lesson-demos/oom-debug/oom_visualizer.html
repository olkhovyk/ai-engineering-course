Без torch.no_grad() — що відбувається:

PyTorch створений для тренування. Кожен раз коли дані проходять через модель, він зберігає computation graph — запис усіх операцій, щоб потім порахувати градієнти через loss.backward().

При тренуванні це потрібно — без графу неможливо оновити ваги.

При інференсі ти ніколи не викликаєш backward(). Але PyTorch цього не знає! Він все одно зберігає граф для кожного запиту. Кожен запит = ще один граф у пам'яті. Пам'ять росте → OOM.

Запит 1: модель(x) → зберіг граф → +8 MB
Запит 2: модель(x) → зберіг граф → +8 MB
Запит 100: модель(x) → ... → +800 MB → 💥 OOM

З torch.no_grad() / inference_mode():

torch.no_grad() каже PyTorch: "не зберігай граф, я не буду викликати backward()".

Результат:
— Немає графу → пам'ять не росте з кожним запитом
— Швидше → менше bookkeeping overhead
— grad_fn = None → тензор не прив'язаний до графу

torch.inference_mode() — строгіша версія (PyTorch 2.0+). Додатково забороняє будь-яке використання результату для градієнтів. Трохи швидше. Рекомендовано для продакшну.

Запит 1: модель(x) → без графу → 28 MB
Запит 2: модель(x) → без графу → 28 MB
Запит 100: модель(x) → без графу → 28 MB ✓

model.eval() — це інше!

model.eval() НЕ вимикає граф. Він вимикає dropout (рандомне обнулення нейронів) і фіксує BatchNorm статистики.
Без нього: той самий вхід → різний результат кожен раз (бачив у панелі Dropout вище).

Потрібні обидва:
model.eval() → стабільний результат (детермінізм)
torch.inference_mode() → стабільна пам'ять (без графу)
