Введи запит — побачиш cosine similarity кожного документа з запитом. Це те, що робить vector DB (Qdrant, Pinecone) під капотом.

Embedding запиту (вектор із 384 чисел)

Кожна клітинка = одне число. Синій = негативне, помаранчевий = позитивне.

Cosine similarity = наскільки "напрямки" двох векторів збігаються (1.0 = ідентичні, 0 = ортогональні)

Що відбувається під капотом model.encode("текст")
Крок 1 Токенізація — текст → token IDs
# "як змінити пароль" → [101, 1210, 23925, 1187, 29745, 29760, 18947, 102] # 101 = [CLS], 102 = [SEP] — спеціальні токени BERT
Крок 2 Forward pass через transformer — token IDs → hidden states
with torch.inference_mode(): # без графу! outputs = model(**encoded) # outputs.last_hidden_state: shape (1, 8, 384) # = 1 речення × 8 токенів × 384-dim вектор кожного
Крок 3 Пулінг — 8 векторів → 1 вектор
# Mean pooling: усереднюємо вектори всіх токенів # (ігноруємо padding через attention_mask) pooled = mean_pooling(hidden_states, attention_mask) # shape: (1, 384) — один вектор для всього речення
Крок 4 Нормалізація — для cosine similarity
normalized = F.normalize(pooled, dim=-1) # Вектор стає unit length (довжина = 1) # Тепер dot product = cosine similarity
Пулінг: як зібрати 1 вектор з N токенів

Transformer видає вектор для КОЖНОГО токена. Нам потрібен один вектор для всього речення. Як?

Порівняння стратегій пулінгу
Mean Pooling стандарт
Середнє всіх токенів (з attention mask).
Використовується у sentence-transformers.
Similarity: 0.721
CLS Token
Перший токен [CLS] — як у BERT.
Не завжди добре для sentence similarity.
Similarity: 0.314
Max Pooling
Максимум по кожному виміру.
Рідко використовується для embeddings.
Similarity: 0.225

Similarity виміряна між "як змінити пароль" і "Для скидання пароля перейдіть у Налаштування"

Батчінг: чому batch=64 в 9x швидше за batch=1

Натисни щоб побачити як throughput зростає з розміром батчу:

Чому батчінг швидший
batch=1: кожен документ окремо → Python loop overhead + GPU idle між запитами
batch=64: 64 документи одночасно → одна матрична операція на GPU

GPU створений для паралельних обчислень. Один документ використовує 1% потужності GPU. 64 документи — 64% потужності. Та сама операція, але GPU не простоює.

Production math (500,000 документів):
batch=1: ~1.3 годин   |   batch=64: ~9 хвилин
Та сама модель, те саме залізо. Тільки batch size змінився.

Production код: embedding-сервіс
from transformers import AutoModel, AutoTokenizer import torch import torch.nn.functional as F MODEL = 'sentence-transformers/all-MiniLM-L6-v2' tokenizer = AutoTokenizer.from_pretrained(MODEL) model = AutoModel.from_pretrained(MODEL).to('cuda') model.eval() # dropout OFF def mean_pooling(hidden, mask): m = mask.unsqueeze(-1).float() return (hidden * m).sum(1) / m.sum(1).clamp(min=1e-9) def embed(texts: list[str]): enc = tokenizer(texts, padding=True, truncation=True, max_length=128, return_tensors='pt').to('cuda') with torch.inference_mode(): # граф OFF h = model(**enc).last_hidden_state pooled = mean_pooling(h, enc['attention_mask']) return F.normalize(pooled, dim=-1) # Використання: embeddings = embed(["як змінити пароль", "скинути пароль"]) similarity = (embeddings[0] @ embeddings[1]).item() # 0.89
Чеклист для production
1. model.eval() — вимкнути dropout (один раз при завантаженні)
2. torch.inference_mode() — кожен виклик (без memory leak)
3. Батчінг: batch_size=64 — 9x швидше за batch=1
4. F.normalize() — L2 нормалізація для cosine similarity
5. padding=True, truncation=True — обробка різної довжини
